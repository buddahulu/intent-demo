# 加拿大稳定币支付钱包 - Figma 原型规格

## 产品概述
- **产品名称**: 待定
- **目标用户**: 加拿大华人用户
- **核心功能**: CAD + 稳定币 + 积分三位一体钱包，支持商户收款
- **平台**: iOS / Android

---

## 页面清单（7 个核心页面）

| 序号 | 页面 | 说明 | 优先级 |
|------|------|------|--------|
| 1 | 首页 | 余额展示、快捷操作、最近交易 | P0 |
| 2 | 充值 | 选择币种/链、显示地址、二维码 | P0 |
| 3 | 转账 | 输入金额、选择收款人、确认 | P0 |
| 4 | 支付 | 扫码、输入金额、选支付方式 | P0 |
| 5 | 积分 | 余额、变现、交易入口 | P1 |
| 6 | 我的 | 设置、安全、KYC、帮助 | P1 |
| 7 | 商户端 | 收款码、交易记录、结算 | P0 |

## 页面详细设计

### 1. 首页

#### 布局结构
```
┌─────────────────────────────┐
│ [头像] 用户名        [通知] │
├─────────────────────────────┤
│                             │
│   总资产 (CAD)              │
│   $12,580.50                │
│                             │
│   [CAD $8,500] [USDT $3,000] [积分 $1,080] │
│                             │
│   [充值]  [转账]  [支付]  [积分] │
│                             │
├─────────────────────────────┤
│ 快捷功能                     │
│ [扫一扫] [收款] [账单]      │
├─────────────────────────────┤
│ 最近交易                     │
│ ┌─────────────────────────┐ │
│ │ 商户A    -$150.00  成功 │ │
│ │ 2分钟前                 │ │
│ ├─────────────────────────┤ │
│ │ 用户B    +$500.00  成功 │ │
│ │ 1小时前                 │ │
│ └─────────────────────────┘ │
└─────────────────────────────┘
```

#### 元素说明

**顶部栏**
- 左侧：用户头像（圆形，40px）+ 用户名（16px Medium）
- 右侧：通知图标（24px），红点提示未读

**总资产卡片**
- 背景：主色渐变 `#1A73E8` → `#1557B0`
- 文字：白色
- 总资产标签：14px Regular
- 金额：32px Bold，SF Mono
- 子余额：横向滚动，标签式展示
- 快捷按钮：4个等宽按钮，图标+文字

**快捷功能**
- 3个功能入口，图标+文字
- 图标：48px 圆形背景

**最近交易**
- 列表形式，最多显示5条
- 每条：头像/图标 + 名称 + 金额 + 状态 + 时间
- 金额：收入绿色，支出黑色

## 交互说明

### 全局交互

| 操作 | 反馈 |
|------|------|
| 点击按钮 | 按钮缩放 0.95，松开后恢复 |
| 页面切换 | 从右向左滑入 |
| 返回 | 从左向右滑出 |
| 下拉刷新 | 顶部加载动画 |
| 上拉加载 | 底部加载动画 |

### 特定交互

| 场景 | 交互 |
|------|------|
| 复制地址 | Toast 提示"已复制" |
| 扫码成功 | 震动 + 页面跳转 |
| 支付成功 | 成功动画 + 音效 |
| 网络错误 | 全局提示 + 重试按钮 |

## 状态设计

### 空状态
- 图标 + 文字说明 + 操作按钮
- 例："暂无交易记录" + "去支付"按钮

### 加载状态
- 骨架屏（首页列表）
- 旋转动画（按钮、页面）

### 错误状态
- 错误图标 + 说明 + 重试按钮
- 例：网络错误、加载失败
